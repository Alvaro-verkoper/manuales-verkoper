INSTRUCCIONES DE INSTALACIÓN:
-----------------------------

1. CREAR REPOSITORIO EN GITHUB:
   - Ve a github.com y crea un nuevo repositorio
   - Nombre sugerido: verkoper-portal-documentos
   - Tipo: Public (para GitHub Pages gratis)

2. SUBIR ARCHIVOS:
   git init
   git add .
   git commit -m "Initial commit - Portal gestión documental"
   git branch -M main
   git remote add origin https://github.com/TU_USUARIO/verkoper-portal-documentos.git
   git push -u origin main

3. ACTIVAR GITHUB PAGES:
   - Ve a Settings → Pages
   - Source: Deploy from a branch
   - Branch: main
   - Folder: / (root)
   - Save

4. ESPERAR DESPLIEGUE:
   - GitHub tomará 2-5 minutos para desplegar
   - El sitio estará en: https://TU_USUARIO.github.io/verkoper-portal-documentos/

COMPLETAR DOCUMENTOS FALTANTES:
-------------------------------

Para cada documento pendiente:

1. Copiar estructura de un documento existente (ej: matrices/estafas.html)
2. Cambiar título, versión, vigencia, aplica a
3. Pegar el contenido oficial en la sección doc-content
4. Actualizar el índice (TOC) con las secciones del documento
5. Agregar/actualizar documentos relacionados

EJEMPLO:

```html
<div class="doc-content">
    <h2 id="section1">1. Título de Sección</h2>
    <p>Contenido de la sección...</p>
    
    <h2 id="section2">2. Siguiente Sección</h2>
    <p>Más contenido...</p>
</div>
```

AGREGAR NUEVOS DOCUMENTOS AL JSON:
----------------------------------

Editar data/docs.json y agregar:

```json
{
  "title": "Nombre del Documento",
  "url": "/ruta/documento.html",
  "section": "protocolos|procedimientos|matrices|otros",
  "tags": ["tag1", "tag2"],
  "updated": "2026-02-05",
  "version": "1.0",
  "area": "Área aplicable",
  "summary": "Resumen breve",
  "critical": true
}
```
